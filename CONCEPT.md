# 盤面 (Banmen) - 構想書

## 背景：なぜこのアプリが必要か

### ワーキングメモリの科学
- 人間が同時に扱える情報は約4つ（±1）
- 体調不良時は実質2つまで低下
- 新しい思考が入ると古いものが脱落 →「なんだっけ？」が発生

### 既存ツールの構造的欠陥
- Todoist / Notion / Obsidian → 溜まる → 開くのが怖くなる → 使わなくなる
- 全てのタスク管理アプリが「記録・保持・蓄積」の方向に進化している
- 結果、管理すること自体が仕事になる

### 盤面の着想
- かるた選手は「覚える力」ではなく「忘れる力」で勝つ
- 前の盤面を捨てて今の盤面だけに集中する
- **データを保持しないことが価値**という逆転の発想

---

## プロダクト定義

> **デジタル時代に、あえて忘れることを許可するツール**

### コア仕様
- **NOWスロット**: 1つだけ。今やっていること
- **NEXTスロット**: 最大3つ。次にやること
- **完了 → 消える**（フェードアウト）
- **履歴なし、統計なし、通知なし**
- **開いて3秒で全部見える**

### 設計原則
1. 引き算の設計 - 機能が少ないことが機能
2. 揮発性 - データを保持しないことが価値
3. 3秒ルール - 開いて3秒で「今やること」がわかる
4. ワーキングメモリの科学 - 4つの制約はバグではなく仕様

### やらないこと
- 履歴・ログ
- 完了数カウント・統計
- 通知・リマインダー
- カテゴリ・タグ・ラベル
- 共有・コラボレーション（初期）

## アーキテクチャ

```
                      ┌─── iPhone / Android / Windows / Mac
  PWA ────────────────┘    (ブラウザ & ホーム画面インストール)

                      ┌─── LINE Bot
  Bot API ────────────├─── Slack App
                      └─── Teams App

                      ┌─── Obsidian Plugin（サイドパネル常駐）
  プラグイン ─────────┘
                      └─── Notion Integration（埋め込み / API連携）

  SNS 集客 ───────────── X / Instagram
```

### なぜPWAか
- 1つのコードで全デバイス対応
- ブラウザで開ける（URL共有だけで使える）
- ホーム画面追加でネイティブアプリと同じ体験
- Windowsデスクトップアプリとしてインストール可能
- オフライン動作可能
- App Store審査不要 → 即公開

## 開発Phase

### Phase 1: PWA（本体）
- Next.js + Service Worker
- 全デバイス対応の土台
- ローカルストレージのみ（サーバー不要）
- デプロイ: Cloudflare Workers (`@opennextjs/cloudflare`)
- ドメイン: Cloudflare Registrar

### Phase 2: Notion Embed
- PWAのURLをNotionに貼るだけ
- 開発作業ほぼなし

### Phase 3: Obsidian Plugin
- サイドパネルに盤面を常駐表示
- Obsidian Community Plugin として公開
- 開発者層への訴求 → 口コミが起きやすい

### Phase 4: Bot連携（LINE / Slack / Teams）
- 共通のBot APIを1つ作り、各プラットフォームにアダプタ接続
- 日常の導線（チャット画面）に盤面を溶け込ませる

### Phase 5: SNS集客（X / Instagram）
- コンセプト発信「タスク管理アプリに疲れた人へ」
- UIスクリーンショットによるビジュアル訴求
