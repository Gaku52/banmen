# Banmen Operating Plan v0.1

## 1. Purpose

Banmen を「思考停止を減らす道具」として成立させるための、実行・計測・収益化の運用設計を定義する。

---

## 2. KPI Definitions

## 2.1 Core KPI

- `A1` (Activation 1): 初回起動から30秒以内に1件追加したユーザー率
- `D7` (Day 7 Retention): 初回利用から7日後に再訪したユーザー率
- `W3` (Weekly 3): 週3回以上、1分以内セッションを行ったユーザー率
- `R1` (Relief 1): 週1の自己申告で「なんだっけが減った」と回答した率

## 2.2 Target (First 90 days)

- A1: `>= 65%`
- D7: `>= 30%`
- W3: `>= 20%`
- R1: `>= 60%`

---

## 3. Event Tracking Spec

最低限、以下のみを計測する。

- `app_open`
- `task_added`
- `task_promoted_to_now`
- `now_completed`
- `undo_used`
- `micro_prompt_shown`
- `micro_prompt_submitted`
- `theme_preview_viewed`
- `theme_purchased`
- `tip_button_tapped`
- `tip_completed`

ルール:
- 個人タスク本文は収集しない
- User ID は匿名IDを基本（任意登録時に連携）
- プライバシーポリシーにイベント収集を明示

---

## 4. Micro Prompt Design

## 4.1 Initial 9 Prompts

着手系:
- 「次の10分で終わる1つは？」
- 「今すぐ手をつけるなら何？」
- 「先延ばししている1つは？」

優先順位系:
- 「今やる価値が一番高いのは？」
- 「終えると一番軽くなるのは？」
- 「今日これだけ進めるなら何？」

詰まり解除系:
- 「迷っている理由を1行で書くと？」
- 「最初の1手だけ書くなら？」
- 「忘れる前に置くべき1つは？」

## 4.2 Display Rules

- NOW が空のときのみ表示
- 1起動で1問のみ
- 同一問いの再表示は24時間空ける
- スキップ可能（強制なし）

## 5. Monetization Spec

## 5.1 基本方針

- **全機能無料**（NOW1/NEXT3、3秒Undo、ローカル保存、同期含む）
- 収益は「見た目カスタム課金」+「投げ銭」の2本立て
- 広告は一切入れない（認知負荷を増やすため哲学に反する）

## 5.2 テーマ課金（メイン収益）

盤面の見た目を変えるテーマを買い切りで販売する。機能は一切変わらない。

カテゴリ:
- **盤面テーマ**: ダーク、和紙風、黒板風、ネオン等（各￥100）
- **フォント変更**: 手書き風、明朝、モノスペース等（各￥100）
- **完了エフェクト**: フェードアウトの演出バリエーション（各￥100）

実装方針:
- CSS変数ベースのテーマシステム
- 購入前にプレビュー可能
- App Store / Google Play のアプリ内課金（買い切り型・非消耗品）

## 5.3 投げ銭（サブ収益）

設定画面の最下部に1行だけ配置:
- 「盤面を気に入った？ ☕ コーヒー1杯おごる（￥200）」
- 金額バリエーション: ￥200 / ￥500 / ￥1,000
- 画面を汚さない。押し売りしない。

## 5.4 やらないこと

- 広告の表示
- サブスクリプション課金
- 機能制限による有料誘導
- 課金しないと使えない機能の追加

## 5.5 Success Metrics

- テーマ購入率: `>= 5%`（DLユーザーのうち1つ以上購入）
- 投げ銭率: `>= 1%`
- 月間テーマ販売数: DL数 × 0.05 × 平均1.5個 を目標

---

## 6. Sync Spec (Future-ready / 無料機能)

## 6.1 Sync Scope

同期するのは「現在盤面のみ」:
- nowTask
- nextTasks (max 3)
- updatedAt

同期しない:
- 完了履歴
- 統計
- 過去ログ

## 6.2 Conflict Resolution

- 基本: last-write-wins
- NOW編集中は3秒の軽い編集ロック
- 衝突時は小さな通知のみ表示（復元履歴は持たない）

## 6.3 Data Expiry

- 長期未使用データは自動失効（例: 7日〜30日）
- 失効期間は検証で調整

## 7. 30/60/90 Day Execution Plan

## 7.1 Day 0-30

- 1画面PWAのMVP公開
- A1/D7計測導入
- 10人ユーザーテスト実施
- マイクロ問い9種の初期検証

## 7.2 Day 31-60

- テーマシステム（CSS変数ベース）実装
- 初期テーマ3種リリース（ダーク、和紙風、黒板風）
- 投げ銭UIを設定画面に配置
- テーマ購入率の初期計測

## 7.3 Day 61-90

- テーマ追加（フォント、完了エフェクト各2種以上）
- X / Product Hunt / Show HN / Reddit で配信
- テーマ購入率・投げ銭率の実測と改善

---

## 8. Go-to-Market Posting Cadence

- X: 週5投稿（開発ログ、10秒デモ、ユーザー学び）
- Product Hunt: ローンチ当日集中対応（コメント返信重視）
- Show HN: 実装詳細と学びを透明に共有
- Reddit: ルール順守でコミュニティごとに文脈最適化

---

## 9. Non-Negotiables

- 1画面原則を崩さない
- NOW1/NEXT3制約を崩さない
- 履歴を価値化しない
- 強制通知で継続を作らない
- プライバシー最小収集を維持

---

## 10. Weekly Review Template

- 今週のA1 / D7 / W3 / R1
- もっとも効いた改善1つ
- 削除した機能・文言1つ
- 来週の実験仮説1つ

## 11. Spec Lock (再相談を減らすための固定仕様)

## 11.1 Priority Order

判断が割れた場合は、以下を上から優先する。

1. 認知摩擦の削減
2. 入力速度
3. シンプルさ
4. 収益性
5. 見た目

## 11.2 Default Decisions

未定義ケースは以下をデフォルト採用する。

- タスク本文の上限: 80文字
- 空文字/空白のみ: 保存不可
- NEXTが3件のとき: 追加不可、ユーザーに短い理由表示
- NOWが埋まっているときの昇格: 入れ替え確認を出す
- 完了操作: 即非表示 + 3秒Undo
- Undoしなかった場合: 完全削除（履歴なし）
- オフライン時: ローカル保存優先、同期は復帰時に再送
- 同期衝突: last-write-wins
- 重大障害時: 同期を一時停止しローカル利用を維持

## 11.3 Copy Lock

以下の文言は原則固定する。

- 「今やることを1つ、次を3つまで置けます。」
- 「終わったら消します。履歴は残しません。」
- 「迷ったら、まず1行だけ書いてください。」

変更条件:
- A/Bテストで指標改善が有意な場合のみ変更

## 11.4 Scope Guard

以下の要求は原則Reject（別Phaseに送る）。

- タグ、カテゴリ、プロジェクト
- 期限、通知、カレンダー連携
- 完了履歴、統計ダッシュボード
- 長文ノート、添付ファイル

## 11.5 Release Gate

リリース前に以下を満たさない場合は出さない。

1. 起動から10秒以内に1件追加できる
2. NOW1/NEXT3制約が壊れない
3. 完了 -> 3秒Undo -> 完全削除が動作する
4. リロード後に盤面が復元される
5. 主要導線でクラッシュしない

## 11.6 Change Policy

- 仕様変更提案は毎週1回だけ受付
- 同時に変更できる主機能は1つまで
- 新機能追加時は「削除する要素」を1つセットで提出

## 11.7 Unknowns Backlog

未確定事項はこの形式で記録する。

- `ID`
- `論点`
- `暫定デフォルト`
- `検証方法`
- `期限`

このフォーマット以外の未確定要素は、実装を止めず暫定デフォルトで進める。
